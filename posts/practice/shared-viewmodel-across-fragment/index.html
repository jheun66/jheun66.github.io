Create a shared ViewModel

model 같은 경우 다른 code들과 분리시켜 놓는게 좋음. 기능별로 패키징!

ViewModel에서 view model data는 public으로 노출하지 않는게 좋음. 대신 backing property를 구현하여 각 private mutable property 별로 public immutable prorperty를 만들어 노출.

편의상 private mutable properties는 _를 붙여줌

private val _quantity = MutableLiveData<Int>(0)
val quantity: LiveData<Int> = _quantity

private val _flavor = MutableLiveData<String>("")
val flavor: LiveData<String> = _flavor

private val _date = MutableLiveData<String>("")
val date: LiveData<String> = _date

private val _price = MutableLiveData<Double>(0.0)
val price: LiveData<Double> = _price

fun setQuantity(numberCupcakes: Int) {
    _quantity.value = numberCupcakes
}

fun setFlavor(desiredFlavor: String) {
    _flavor.value = desiredFlavor
}

fun setDate(pickupDate: String) {
    _date.value = pickupDate
}

Use Kotlin property delegate

코틀린에서는 각각의 mutable proprerty는 기본 getter와 setter 함수를 자동으로 생성. property에 값을 할당하거나 값을 읽을 때 호출됨. immutable property는 getter만 생성되고 호출됨.

Property delegation은 이러한 getter-setter에 대한 책임을 다른 클래스에 이양할 수 있음.

// Syntax for property delegation
var <property-name> : <property-type> by <delegate-class>()

하나의 액티비티에 있는 다른 Fragement가 하나의 ViewModel을 공유하기 위해서는 다음과 같이 각 프래그먼트에 Viewmodel을 선언할 때 acitivityViewModels를 delegate-class로 해주면 됨

private val sharedViewModel: OrderViewModel by activityViewModels()

Use ViewModel with data binding

viewModel을 data binding과 함께 사용하려면 xml 파일에 data 태그를 추가함.

<layout ...>

    <data>
        <variable
            name="viewModel"
            type="com.example.cupcake.model.OrderViewModel" />
    </data>

    <ScrollView ...>

    ...

Fragment의 onViewCraeted 메소드 안에 세팅

binding?.apply {
    viewModel = sharedViewModel
    ...
}

apply는 scope function, 다른 function도 알아두어야함

바인딩한 viewModel을 xml 파일에서 사용할 때는 다음과 같이 @ symbol로 시작하고 {}로 감싸줌

<RadioButton
     android:id="@+id/coffee"
     ...
     android:checked="@{viewModel.flavor.equals(@string/coffee)}"
     .../>

클릭 이벤트에 대한 리스너는 다음과 같이 람다함수를 이용할 수 있음

<RadioButton
    android:id="@+id/coffee"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content"
    android:onClick="@{() -> viewModel.setFlavor(@string/coffee)}"
    android:checked="@{viewModel.flavor.equals(@string/coffee)}"
    android:text="@string/coffee" />

Set Lifecycle owner to observe LiveData

Data Binding에서 layout 파일에서 binding expression에 의해 observable 값이 변했을 때 UI 요소를 변화시키기 위해서는 binding.lifecycleOwner를 viewLifecycleOwner로 연결해주어야한다.

binding?.apply {   
    lifecycleOwner = viewLifecycleOwner
    ...
}

Format price with LiveData transformation

Transformations.map을 통해 포맷이 다른 값을 매핑하여 변경해줄 수 있음

private val _price = MutableLiveData<Double>()
val price: LiveData<String> = Transformations.map(_price) {
   NumberFormat.getCurrencyInstance().format(it)
}
