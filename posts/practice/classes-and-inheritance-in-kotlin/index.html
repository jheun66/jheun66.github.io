Classes and inheritance in Kotlin

Posted by Eun JongHyeok on July 25, 2021 Kotlin

개요

이 글은 안드로이드 개발자 사이트의 내용을 따라해보면서 작성하였습니다. 코틀린 관련 자세한 내용은 코틀린 사이트를 참고하면 좋을 것 같습니다.

Classes

keyword

class

Declaration

name, header, body로 이루어지며 header와 body는 생략이 가능

Constructor

keyword는 constructor로, 하나의 primary constructor와 하나 이상의 secondary constructor를 가질 수 있음

primary constructor

class MyClass constructor(className: String)
class MyClass(className: String)

클래스 이름과 optional parameters 뒤에 올 수 있으며 annotations나 visibility modifiers(자바의 접근 제한자와 유사)가 없으면 생략 가능

init 구문과 초기화

primary constructor에는 어떠한 코드도 포함할 수 없기 때문에 초기화 코드는 init 구문안에 작성

class InitOrderDemo(name: String) {
    val firstProperty = "First property: $name".also(::println)
    
    init {
        println("First initializer block that prints ${name}")
    }
    
    val secondProperty = "Second property: ${name.length}".also(::println)
    
    init {
        println("Second initializer block that prints ${name.length}")
    }
}

property 초기화 사이에서 같은 우선순위로 수행되며 초기화의 매개변수는 property 초기화 단계에서 사용될 수 있음

간결하게 선언에서 바로 기본값을 넣을 수 있음

secondary constructor

클래스 바디에 constructor를 붙여서 선언, primary constructor가 있다면 this를 이용한 Delegation(위임) 필요

class Person(val name: String) {
    var children: MutableList<Person> = mutableListOf()
    constructor(name: String, parent: Person) : this(name) {
        parent.children.add(this)
    }
}

이러한 Delegation(위임)에 의해 secondary constructor 내부 코드가 진행되기 전에 초기화 구문과 property 초기화가 이루어짐, 이는 primary constructor가 없더라도 위임은 암시적으로 일어남

Inheritance

코틀린의 모든 클래스의 부모 클래스는 Any 클래스, Any 클래스는 equals(), hashcode(), toString() 함수를 가짐

open

클래스가 상속가능하다는 의미로 open 키워드 사용, 코틀린 클래스의 자바나 C++과 다른게 제한자의 기본값이 final이며 생략되어 있음, 메소드에도 동일하게 사용

override

자식클래스에서 부모클래스의 메소드를 override할 때 사용, override는 open 제한자처럼 계속 상속받을 수 있으므로 해당 메소드가 더이상 상속되길 원하지 않으면 final을 앞에 붙여줌

open class Shape {
    open fun draw() { /*...*/ }
    fun fill() { /*...*/ }
}

class Circle() : Shape() {
    override fun draw() { /*...*/ }
}

open class Rectangle() : Shape() {
    final override fun draw() { /*...*/ }
}

super

동일한 메소드명을 상속 받을 경우 다음과 같이 모호함을 없애줌

open class Rectangle {
    open fun draw() { println("draw rectangle!") }
}

interface Polygon {
    fun draw() { println("draw polygon!") } // interface members are 'open' by default
}

class Square() : Rectangle(), Polygon {
    // The compiler requires draw() to be overridden:
    override fun draw() {
        super<Rectangle>.draw() // call to Rectangle.draw()
        super<Polygon>.draw() // call to Polygon.draw()
    }
}

fun main() {
    val square = Square()
    square.draw()
}

Adopt good coding practices

fun main() {
    val squareCabin = SquareCabin(6)
       
    with(squareCabin) {
        println("\nSquare Cabin\n============")
        println("Capacity: ${capacity}")
        println("Material: ${buildingMaterial}")
        println("Has room? ${hasRoom()}")
    }
}

abstract class Dwelling(private var residents: Int) {
    abstract val buildingMaterial: String
    abstract val capacity: Int
       
    fun hasRoom(): Boolean {
       return residents < capacity
   }
}

class SquareCabin(residents: Int) : Dwelling(residents) {
    override val buildingMaterial = "Wood"
    override val capacity = 6
}

with

with 키워드를 이용하여 동일한 instance의 멤버에 접근하여 코드를 간단한게 할 수 있음
