Fragments and the fragment lifecycle

fragment 역시 activity처럼 라이프사이클을 가지고 있음

Fragment lifecycle

5가지 Lifecylce.State 를 가짐

  • INITALIZED
  • CREATED
  • STARTED
  • RESUMED
  • DESTROYED

또한 Activity처럼 각 Lifecyle에 대응하는 이벤트 함수들이 있음

  • onCreate()

프래그먼트가 초기화되고 CREATED 상태에 들어갈 때

  • onCreateView()

레이아웃을 inflate하는 메소드

  • onViewCreated()

뷰가 만들어진 뒤 호출, 일반적으로 특정 뷰를 바인딩함

  • onStart()

프레그먼트가 STARTED 상태에 들어갈 때

  • onResume()

프레그먼트가 RESUMED 상태에 들어갈 때, 포커스를 가짐

  • onPause()

프레그먼트가 다시 STARTED 상태에 들어갈 때, 포커스를 뺏기지만 유저한테는 보임

  • onStop()

프레그먼트가 다시 CREATED 상태에 들어갈 때, 객체가 생성되었지만 더이상 화면에 보이지 않음

  • onDestroyView()

프레그먼트가 DESTROYED 상태에 들어가기 직전에 호출, 뷰가 메모리에서 지워졌지만 프레그먼트 객체는 남아 있음

  • onDestroy()

프레그먼트가 DESTROTED 상태에 들어갈 때

액티비티의 라이프사이클과 비교할 때 주의

액티비티에서는 onCreate() 함수에서 레이아웃을 inflate하고 뷰를 바인딩했지만 프레그먼트에서는 뷰가 생성되기 전에 onCraete()를 호출함

Implement LetterListFragment

  1. 바인딩 객체 만들기

액티비티 바인딩 객체처럼 Android Studio가 각 레이아웃 파일마다 해당 클래스를 생성

private var _binding: FragmentLetterListBinding? = null

FragmentLetterListBinding? 타입이 nullable인 이유는 초기화값이 null 값을 가지기 때문.

  1. binding이라는 새로운 프로퍼티 만들기

onCreateView()이후에는 _binding에 null값이 들어가지 않을텐데 매번 _binding에 ?를 붙이기 번거롭기 때문에 !!를 이용해 ?없이 접근 가능하도록 할 수 있음.

private val binding get() = _binding!!

get() 는 binding이 get-only 프로터티라는 의미

변수명 앞에 _가 있는 경우 직접 접근해서 사용하지 않는다는 것을 의미함

  1. onCreate() 오버라이드 후 setHasOptionsMenu() 추가
override fun onCreate(savedInstanceState: Bundle?) {
   super.onCreate(savedInstanceState)
   setHasOptionsMenu(true)
}
  1. onCreateView()에서 inflate후 view 반환
override fun onCreateView(
   inflater: LayoutInflater, container: ViewGroup?,
   savedInstanceState: Bundle?
): View? {
   _binding = FragmentLetterListBinding.inflate(inflater, container, false)
   val view = binding.root
   return view
}
  1. 리사이클러뷰 프로퍼티 생성
private lateinit var recyclerView: RecyclerView
  1. onViewCreated()에서 리사이클러뷰 프로퍼티에 값 세팅, 레이아웃을 선택하는 메소드 추가
override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
   recyclerView = binding.recyclerView
   chooseLayout()
}
  1. onDestroyView()에서 _binding 값을 잊지않고 null로 초기화
override fun onDestroyView() {
   super.onDestroyView()
   _binding = null
}
  1. 프레그먼트의 onCreateOptionsMenu()에는 리턴값이 없고 액티비티처럼 menuInflater라는 글로벌 프로퍼티가 없어 menuInflater를 매개변수로 넘겨주어야함
override fun onCreateOptionsMenu(menu: Menu, inflater: MenuInflater) {
   inflater.inflate(R.menu.layout_menu, menu)

   val layoutButton = menu.findItem(R.id.action_switch_layout)
   setIcon(layoutButton)
}
  1. 나머지는 액티비티에서 만들었던 함수 그대로 사용, this 대신 context를 넘겨주어야함
private fun chooseLayout() {
   when (isLinearLayoutManager) {
       true -> {
           recyclerView.layoutManager = LinearLayoutManager(context)
           recyclerView.adapter = LetterAdapter()
       }
       false -> {
           recyclerView.layoutManager = GridLayoutManager(context, 4)
           recyclerView.adapter = LetterAdapter()
       }
   }
}

private fun setIcon(menuItem: MenuItem?) {
   if (menuItem == null)
       return

   menuItem.icon =
       if (isLinearLayoutManager)
           ContextCompat.getDrawable(this.requireContext(), R.drawable.ic_grid_layout)
       else ContextCompat.getDrawable(this.requireContext(), R.drawable.ic_linear_layout)
}

override fun onOptionsItemSelected(item: MenuItem): Boolean {
   return when (item.itemId) {
       R.id.action_switch_layout -> {
           isLinearLayoutManager = !isLinearLayoutManager
           chooseLayout()
           setIcon(item)

           return true
       }

       else -> super.onOptionsItemSelected(item)
   }
}

Fragments and the Navigation Component

Navigation Component

Navigation Component에 대한 최신 가이드는 가이드 문서를 참고하는게 좋을 것 같음

  • Navigation Graph: 앱의 Navigation을 시각적으로 확인할 수 있도록 도와주는 XML File. 각각의 Fragment와 Activity에 상응하는 화면으로 목적지를 지정할 수 있음.
  • NavHost: 액티비티의 Navigation Graph에 목적지를 띄워주는데 사용. 프레그먼트를 결정하면 Navhost의 목적지가 업데이트
  • NavController: NavHost에서 보여지는 목적지를 컨트롤하는 객체. Navigation Component를 사용할 경우, NavController의 nagivate()메소드를 호출하여 프레그먼트를 바꿔줌. NavController는 공통된 동작, 예를들어 시스템 “up” 버튼을 누를 경우 이전 프레그먼트로 가도록 할 수 있음

Navigation Component를 사용하기 위한 작업 필요

Navigation Dependency

  1. 프로젝트 레벨 build.gradle에 nav_version 세팅
buildscript {
    ext {
        appcompat_version = "1.2.0"
        constraintlayout_version = "2.0.2"
        core_ktx_version = "1.3.2"
        kotlin_version = "1.3.72"
        material_version = "1.2.1"
        nav_version = "2.3.1"
    }

    ...
  1. 앱 레벨 build.gradle에 dependencies group 추가
implementation "androidx.navigation:navigation-fragment-ktx:$nav_version"
implementation "androidx.navigation:navigation-ui-ktx:$nav_version"

Safe Args Plugin

액티비티 전환에서 intent에 putExtra()를 통해 정보를 넘겼던 것처럼 Navigation Component에서는 Safe Args을 사용함. 이를 사용하기 위해서는 마찬가지로 build.gradle에 추가 작성 필요.

  1. 프로젝트 레벨 build.gradle에 classpath 추가
classpath "androidx.navigation:navigation-safe-args-gradle-plugin:$nav_version"
  1. 앱 레벨 build.gradle에 plugins 추가
plugins {
    id 'com.android.application'
    id 'kotlin-android'
    id 'kotlin-kapt'
    id 'androidx.navigation.safeargs.kotlin'
}

Using the Navigation Graph

Use FragmentContainerView in MainActivity

프레그먼트들을 띄워줄 Activity에 FragmentContainerView 추가

<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <androidx.fragment.app.FragmentContainerView
        android:id="@+id/nav_host_fragment"
        android:name="androidx.navigation.fragment.NavHostFragment"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:defaultNavHost="true"
        app:navGraph="@navigation/nav_graph"
        />
</FrameLayout>

class MainActivity : AppCompatActivity() {
    private lateinit var navController: NavController

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)

        val binding = ActivityMainBinding.inflate(layoutInflater)
        setContentView(binding.root)

        val navHostFragment = supportFragmentManager
            .findFragmentById(R.id.nav_host_fragment) as NavHostFragment
        navController = navHostFragment.navController

        setupActionBarWithNavController(navController)
    }

    override fun onSupportNavigateUp(): Boolean {
        return navController.navigateUp() || super.onSupportNavigateUp()
    }
}

Set Up the Navigation Graph

  • File > New > Android Resource File에서 ResourceType을 Navigation으로 하여 생성
  • 원하는 프래그먼트 추가하고 start로 사용할 프레그먼트 지정
  • 화살표로 이어주면 action이 생성됨
  • Destination Fragment에 Arguments를 넣어줄 수 있음
  • Fragment에서 label도 지정가능
