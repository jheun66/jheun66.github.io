What is LiveData

LiveData는 lifecycle을 인지하는 관찰가능한 데이터홀더 클래스

LiveData의 특징

  • data를 담고있음 : 어떤 타입의 데이터도 사용할 수 있는 wrapper
  • 관찰가능함 : 이것이 의미하는 것은 LiveData 객체가 담고있는 데이터가 변할 때 관찰자가 알 수 있음
  • lifecycle-aware : active lifecycle 상태(STARTED, RESUMED)인 옵저버들을 업데이트

MutableLiveData

LiveData에도 Mutable 버전이 있음

private val _currentScrambledWord = MutableLiveData<String>()
val currentScrambledWord: LiveData<String>
   get() = _currentScrambledWord

Type에는 _currentScrambledWord.value = String(tempWord)와 같이 .value로 접근

Activity나 Fragement에서는 onCreate, onCreateView에서 observer를 붙여줌

// Observe the currentScrambledWord LiveData.
viewModel.currentScrambledWord.observe(viewLifecycleOwner, {
    newWord ->
        binding.textViewUnscrambledWord.text = newWord
})

Use LiveData with data binding

Refresher for View binding

View Binding은 코드에서 쉽게 뷰에 접근할 수 있는게 특징, 각 XML layout file 마다 바인딩 클래스가 생성.

단방향 바인딩이라 layout file에서는 앱 데이터에 접근을 못함.

Data Binding

layout file에서 UI요소와 data를 결합하도록 할 수 있음. 간단하게 양방향 바인딩

android:text="@{gameViewModel.currentScrambledWord}"

@{} 문법을 사용하여 앱 데이터를 바인딩

Data Binding의 장점은 액티비티에서 UI framework를 호출들을 제거할 수 있다는 점. 이를 통해 앱 성능 향상, 메모리 누수 null pointer execption등을 방지

Change view binding to data binding

app 수준의 build.gradle에서 dataBinding을 위한 작업

plugins {
    id 'com.android.application'
    id 'kotlin-android'
    id 'kotlin-kapt' // 추가
}

android {
    //...
    buildFeatures {
       dataBinding = true // 추가
    }
}

Convert layout file to data binding layout

<layout xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:app="http://schemas.android.com/apk/res-auto"
   xmlns:tools="http://schemas.android.com/tools">

   <data>

   </data>

   <ScrollView
   android:layout_width="match_parent"
   android:layout_height="match_parent">

   <androidx.constraintlayout.widget.ConstraintLayout
         ...
   </androidx.constraintlayout.widget.ConstraintLayout>
   </ScrollView>
</layout>

기존 Layout file을 <layout> 태그로 한번 감싸기

<data></data> 추가

Fragment에서는 DataBindingUtil을 이용해 inflate

binding = DataBindingUtil.inflate(inflater, R.layout.game_fragment, container, false)

Add data binding variables

layout에 변수 선언

<data>
      <variable
          name="gameViewModel"
          type="com.example.android.unscramble.ui.game.GameViewModel" />
      <variable
          name="maxNoOfWords"
          type="int" />
  </data>

onCraeteView에서 layout 변수 초기화

override fun onViewCreated(view: View, savedInstanceState: Bundle?) {
   super.onViewCreated(view, savedInstanceState)

   binding.gameViewModel = viewModel

   binding.maxNoOfWords = MAX_NO_OF_WORDS
     // Specify the fragment view as the lifecycle owner of the binding.
   // This is used so that the binding can observe LiveData updates
   binding.lifecycleOwner = viewLifecycleOwner
...
}

Use binding expressions

view에 바로 값을 할당할 수 있음

<TextView
   android:id="@+id/textView_unscrambled_word"
   ...
   android:text="@{gameViewModel.currentScrambledWord}"
   .../>

밑의 코드는 불필요해짐

viewModel.currentScrambledWord.observe(viewLifecycleOwner,
   { newWord ->
       binding.textViewUnscrambledWord.text = newWord
   })

app의 변수뿐만 아니라 resource 매개변수에 값을 넣어서 넘겨줄수있음

<TextView
   android:id="@+id/score"
   ...
   android:text="@{@string/score(gameViewModel.score)}"
   ... />
