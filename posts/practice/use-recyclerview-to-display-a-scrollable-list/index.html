개요

이 글은 안드로이드 개발자 사이트의 내용을 따라해보면서 작성하였습니다

Data Class

package me.amos.affirmations.model

data class Affirmation(val stringResourceId: Int) {
}
  • data, Data Class를 만드는 키워드
  • primary constructor 필수
  • abstract, open, sealed, inner 클래스 불가능

primary constructor에 포함된 프로터피로만 toString(), equals(), hashCode(), copy() 적용

즉, 해당 프로퍼티만 같으면 같은 data로 취급

RecyclerView

<androidx.recyclerview.widget.RecyclerView
        android:id="@+id/recycler_view"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        app:layoutManager="LinearLayoutManager"

  • app:layoutManager

RecyclerView는 LayoutManger를 통해 선형 리스트나 그리드처럼 다양한 형태로 아이템을 띄워줌

Implement an Adapter for the RecyclerView

Adapter

RecyclerView에서 사용할 수 있는 데이터로 변환, 예를 들어 화면에 3개의 아이템만 띄워줄 경우 RecyclerView는 Adapter에 3개의 아이템 뷰만 준비하도록함

Create a layout for items

RecylerView에 띄워줄 ItemView 생성

<?xml version="1.0" encoding="utf-8"?>
<TextView xmlns:android="http://schemas.android.com/apk/res/android"
    android:id="@+id/item_title"
    android:layout_width="wrap_content"
    android:layout_height="wrap_content" />

Create an ItemAdapter class

package me.amos.affirmations.adapter

import android.content.Context
import me.amos.affirmations.model.Affirmation

class ItemAdapter(private val context: Context, private val dataset: List<Affirmation>) {
}

Create a ViewHolder

RecyclerView를 대신해 itemView와 직접 상호작용하는 ViewHolder 생성, 말그대로 뷰가 담겨있는 통?

class ItemAdapter(private val context: Context, private val dataset: List<Affirmation>) {

    class ItemViewHolder(private val view: View) : RecyclerView.ViewHolder(view) {
        val textView: TextView = view.findViewById(R.id.item_title)
    }
}

클래스 안의 클래스를 nested class라 함, ItemAdapter에서만 ItemViewHolder를 사용하기에 이런 식으로 작성

상속받게되면 만드시 구현해야하는 3가지 메소드

  • Implement getItemCount()

더욱 간결하게 표현가능

override fun getItemCount() = dataset.size
  • Implement onCreateViewHolder()

새로운 ViewHolder를 만들 때 Layout Manager에 의해 호출

override fun onCreateViewHolder(parent: ViewGroup, viewType: Int): ItemViewHolder {
        // create a new view
        val adapterLayout = LayoutInflater.from(parent.context)
            .inflate(R.layout.list_item, parent, false)

        return ItemViewHolder(adapterLayout)
    }

parent는 RecyclerView, viewType은 하나의 RecyclerView에 다른 타입의 뷰 리스트를 담고있을 때 사용

LayoutInflater.from(parent.context)

RecyclerView의 Context에서 한번 LayoutInflater 객체를 생성하면 해당 객체를 계속 사용

inflate의 3번째 매개변수 attachToRoot에 대한 오해

false로하면 개발자가 원하는 시간에 붙여줄수 있음

  • Implement onBindViewHolder()

생성한 ViewHolder에 데이터를 바인딩

override fun onBindViewHolder(holder: ItemViewHolder, position: Int) {
    val item = dataset[position]
    holder.textView.text =  context.resources.getString(item.stringResourceId)
}

Modify the MainActivity to use a RecyclerView

class MainActivity : AppCompatActivity() {
    lateinit var binding: ActivityMainBinding

    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        binding = ActivityMainBinding.inflate(layoutInflater)
        setContentView(binding.root)

        val myDataset = Datasource().loadAffirmations()

        val recyclerView = binding.recyclerView
        recyclerView.adapter = ItemAdapter(this, myDataset)

        recyclerView.setHasFixedSize(true)
    }
}
  • recyclerView.setHasFixedSize(true)

RecyclerView의 레이아웃 크기가 변하지 않는 것을 알면 성능향상을 위해서 사용
