Display an internet image

웹 URL으로부터 사진을 띄우는 것은 간단하게 들리지만 꽤 많은 작업이 필요. 이미지를 다운받고 내부에 저장하고 압축된 포맷을 이미지로 디코딩 해야함. 이미지는 메모리내의 캐시나 저장소기반의 캐시, 혹은 둘다에 캐싱되어야하고 모든 동작들이 UI response를 위해 우선순위가 낮은 쓰레드에서 동작되어야함. 또한 네트워크나 CPU 성능을 위해서 여러 이미지를 동시에 가져오고 디코딩해야함.

다행이도 Coil 라이브러리를 사용하면 이러한 많은 일을 직접할 필요가 없음.

Coil은 기본적으로 2가지가 필요함

  • 보여줄 이미지의 URL
  • 이미지를 보여줄 ImageView

Add Coil dependency

coil 라이브러리는 mavenCentral repository에서 호스트되고 이용가능하므로 프로젝트 레벨의 build.gradle에 mavenCentral() 추가

repositories {
   google()
   jcenter()
   mavenCentral()
}

앱 레벨의 build.gradle에 라이브러리 추가

// Coil
implementation "io.coil-kt:coil:1.1.1"

Use Binding Adapters

Binding Adpaters는 뷰의 커스텀 프로퍼티를 위한 커스텀 setter를 만들기 위해 사용되는 주석화된 메소드

XML에서 일반적으로 attribute를 세팅하는 코드는 android:text="Text" 처럼 사용되는데 이때 안드로이드 시스템은 자동적으로 안드로이드 프레임워크에서 setText(String: text)를 호출. 비슷한 방식으로 어탭터 바인딩을 사용하면 커스텀 속성과 로직을 설정할 수 있음

Create a binding adapter and use Coil

@BindingAdapter("imageUrl")
fun bindImage(imgView: ImageView, imgUrl: String?) {
    imgUrl?.let {
        // Load the image in the background using Coil.
        }
    }
}

@BindingAdapter Annotation의 매개변수가 커스텀 속성의 이름이됨

let scope function

let은 코틀린 Scope 함수 중 하나로 코드블록을 object의 context내에서 실행할 수 있도록 해줌.

호출 체인의 결과에 하나 이상의 함수를 호출할 때 사용. 주로 널이 아닌 객체에 작업을 수행하기 위해 안전 호출 연산자(?.)와 함께 사용

Display a grid of images with a RecyclerView

Grid Layout

RecyclerView를 위한 GridLayoutManager는 스크롤 가능한 격자형태로 데이터를 담아줌

Add Recyclerview

<androidx.recyclerview.widget.RecyclerView
    android:id="@+id/photos_grid"
    android:layout_width="0dp"
    android:layout_height="0dp"
    app:layoutManager="androidx.recyclerview.widget.GridLayoutManager"
    app:layout_constraintBottom_toBottomOf="parent"
    app:layout_constraintLeft_toLeftOf="parent"
    app:layout_constraintRight_toRightOf="parent"
    app:layout_constraintTop_toTopOf="parent"
    app:spanCount="2"
    android:padding="6dp"
    tools:itemCount="16"
    tools:listitem="@layout/grid_view_item"/>

Material 가이드를 참고하면 다음과 같이 밖에는 8dp, 내부는 4dp를 사용하는 것을 권장

<ImageView
    android:id="@+id/mars_image"
    android:layout_width="wrap_content"
    android:layout_height="170dp"
    android:scaleType="fitXY"
    android:adjustViewBounds="true"
    android:padding="2dp"
    app:imageUrl="@{photos.imgSrcUrl}"
    tools:src="@tools:sample/backgrounds/scenic"/>

그리드 내부 이미지에 2dp씩 주었기 때문에 밖에만 6dp padding 추가하면 깔끔해짐

Add the photo grid adapter

ListAdapter

class PhotoGridAdapter : ListAdapter<MarsPhoto,
        PhotoGridAdapter.MarsPhotoViewHolder>(DiffCallback){

    companion object DiffCallback : DiffUtil.ItemCallback<MarsPhoto>() {
        override fun areItemsTheSame(oldItem: MarsPhoto, newItem: MarsPhoto): Boolean {
            return oldItem.id == newItem.id
        }

        override fun areContentsTheSame(oldItem: MarsPhoto, newItem: MarsPhoto): Boolean {
            return oldItem.imgSrcUrl == newItem.imgSrcUrl
        }
    }

    override fun onCreateViewHolder(
        parent: ViewGroup,
        viewType: Int
    ): PhotoGridAdapter.MarsPhotoViewHolder {
        return MarsPhotoViewHolder(GridViewItemBinding.inflate(
            LayoutInflater.from(parent.context)))
    }

    override fun onBindViewHolder(holder: PhotoGridAdapter.MarsPhotoViewHolder, position: Int) {
        val marsPhoto = getItem(position)
        holder.bind(marsPhoto)
    }

    class MarsPhotoViewHolder(private var binding: GridViewItemBinding): RecyclerView.ViewHolder(binding.root) {
        fun bind(marsPhoto: MarsPhoto) {
            binding.photo = marsPhoto
            binding.executePendingBindings()
        }
    }
}

ListAdapter는 RecyclerView.Adapter 클래스의 서브클래스로 RecyclerView내의 리스트 데이터를 표현하고 백그라운드 스레드에서 차이를 계산해줌.

DiffUtil을 사용하여 전체 리스트를 갱신하지 않고 추가된 아이템이나 제거된 아이템만 처리해줌.

executePendingBindings()은 즉시 업데이트하기위해 사용

Add the binding adapter and connect the parts

@BindingAdapter("listData")
fun bindRecyclerView(recyclerView: RecyclerView,
                     data: List<MarsPhoto>?) {
    val adapter = recyclerView.adapteras PhotoGridAdapter
    adapter.submitList(data)
}

xml의 recyclerView에 app:listData="@{viewModel.photos}" 추가

android:clipToPadding="false"는 바닥에 패딩 생기는거 없애줌
