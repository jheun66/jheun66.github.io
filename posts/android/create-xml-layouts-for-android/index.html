Create XML layouts for Android

  • 개요
  • Read and underStand XML

개요

이 글은 안드로이드 개발자 사이트의 내용을 따라해보면서 작성하였습니다.

Read and underStand XML

<?xml version="1.0" encoding="utf-8"?>
<androidx.constraintlayout.widget.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".MainActivity">

    <!--    TextView Comment 추가   -->
    <TextView
        android:layout_width="wrap_content"
        android:layout_height="wrap_content"
        android:text="Hello World!"
        app:layout_constraintBottom_toBottomOf="parent"
        app:layout_constraintLeft_toLeftOf="parent"
        app:layout_constraintRight_toRightOf="parent"
        app:layout_constraintTop_toTopOf="parent" />

</androidx.constraintlayout.widget.ConstraintLayout>
  • XML

eXtensible Markup Language의 약자

  • <?xml version=”1.0” encoding=”utf-8”?>

이 파일일 xml 파일이라는 것을 나타냄, 모든 xml이 가지고 있는 것은 아님

  • xmlns

XML namespace의 약자

  • xmlns:android, xmlns:app, xmlns:tools

XML namespace인 android, app, tools에 대한 스킴, 용어가 정의된 경로, 이게 있어야 해당 네임스페이스를 사용할 수 있음

  • 주석 사용

Build the layout in XML

  • **layout_constraint_toOf**

ConstraintLayout에서 Constraint를 걸어주는 방식, Source나 Target에 Left, Right 대신 Start, End 사용할 수 있음, 왼쪽에서 오른쪽으로 쓰는 언어와 오른쪽에서 왼쪽으로 쓰는 언어 모두에게 적절한 layout을 적용하고자 하면 사용

  • A resource ID

Layout Editor에서 View를 생성하면 안드로이드 스튜디오에서 자동으로 생성, 직접 명시적으로 선언할 경우 @+id/ 를 붙이고 이름을 넣어주면 됨, 코드에서 참조 할때는 *R..* 형태로 사용

  • 0dp

ConstraintLayout에서는 match_parent를 사용할 수 없고 대신 0dp를 사용
